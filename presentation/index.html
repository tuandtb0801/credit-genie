The Personal Loan path is organized around one Main Agent: the Orchestrator.

It owns planning, context management, timing, and the visible SSE stream. It delegates scoped work to Eligibility, Affordability, and Risk subagents. Eligibility remains rules-only; Affordability and Risk use bounded reasoning and can challenge each other.

The subagents return assessments to the Orchestrator. The Orchestrator then calls deterministic scoring and explanation tools, records the full lineage, and escalates unresolved disagreement to REFER.

BNPL keeps the same Orchestrator-and-subagent structure, but switches every delegated assessment to rules-only mode.

The Orchestrator fans eligibility, affordability, and risk checks out in parallel. These subagents do not call an LLM and do not run a collaboration round. They return deterministic results immediately.

The Orchestrator passes those results through scoring and template explanation tools, then writes the same audit-lineage format—all within the two-second budget.

07 / SHARED ARCHITECTURE

From request to traceable decision

Intelligence at the edges.
Control at the center.

The request enters with an applicant and a product.

First, the evidence collector fetches the mock bureau, income and transaction data, exposure, policy, and historic outcomes. The preparation layer masks sensitive fields, normalizes the evidence, builds a flat packet, and detects useful signals such as irregular deposits.

The mode router sends Personal Loans to the full path and BNPL to the fast path.

For Personal Loans, the eligibility gate applies deterministic minimum criteria first. Affordability and Risk then assess scoped evidence. The orchestrator applies hard rules, weights, thresholds, and consensus checks, then generates explanations from the same lineage.

For BNPL, the rules engine evaluates eligibility, affordability, risk, exposure, and stacking directly. No LLM is on that path.

Every result returns a decision, evidence references, timing, confidence, and audience-specific explanation. The Personal Loan path streams stages and agent messages to the UI.

08 / BOUNDED AGENCY

Agentic ≠ autonomous approval

Narrow judgment.
Deterministic control.

  • Agents assess scoped evidence.
  • Tools own rules, scoring, and thresholds.
  • Conflict escalates to REFER.

Agentic does not mean letting one model decide everything.

Credit Genie has three decisioning roles. Eligibility is a rules-only gate. Affordability reasons about DTI, income stability, and bank-statement patterns. Risk checks delinquency, exposure, BNPL stacking, and fraud signals.

Risk can challenge Affordability. Affordability responds with a re-evaluation. If disagreement remains, the system escalates to REFER.

Python remains the deterministic orchestrator. It controls sequencing, timeouts, scoring, weights, thresholds, and consensus. Agents handle the narrow reasoning tasks where judgment adds value.

That separation gives us intelligence with reproducibility.
