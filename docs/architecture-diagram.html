Two execution paths, one policy

Personal loans and BNPL can't share a workflow — one needs room to reason, the other needs to finish before checkout does. Both read the same policy file and the same hard rules; only the depth of assessment changes.

Decision logic

Hard rules always run first and can end the decision immediately. What survives them goes through a consensus check before it ever reaches a weighted score — disagreement between agents is a first-class reason to refer, not a bug to average away.

Policy lifecycle

The pitch is "no code deployment" — but that only holds up if every change is still tested, approved, and reversible. Every stage below is a checkpoint, not a formality.

Draft
analyst edits YAML
Validate
schema + conflicts
Simulate
re-run past decisions
Approve
authorized sign-off
Activate
effective_date set
Monitor
watch approval drift
Rollback — if approval rate drifts after activation, the previous version reverts instantly and monitoring continues on it. No re-deployment, no waiting on engineering.
